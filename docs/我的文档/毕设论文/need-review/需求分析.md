# 需求分析

本系统是一个医疗陪诊服务管理平台，主要分为管理后台和小程序前端两部分。系统旨在为患者提供挂号预约、医疗陪诊等一站式医疗服务解决方案。

## 系统角色

```mermaid
graph TD
    A[系统角色] --> B[管理员]
    A --> C[普通用户]
    A --> D[陪诊人员]

    B --> B1[系统管理员]
    B --> B2[运营管理员]

    C --> C1[患者]
    C --> C2[家属]

    D --> D1[专业陪诊]
    D --> D2[普通陪诊]
```

## 核心业务流程

```mermaid
sequenceDiagram
    participant 用户
    participant 小程序
    participant 后台系统
    participant 陪诊人员

    用户->>小程序: 注册/登录
    用户->>小程序: 预约挂号/陪诊服务
    小程序->>后台系统: 提交订单信息
    后台系统->>陪诊人员: 分配陪诊任务
    陪诊人员->>用户: 提供陪诊服务
    用户->>小程序: 完成服务评价
    小程序->>后台系统: 记录服务评价
    后台系统->>后台系统: 数据分析与统计
```
